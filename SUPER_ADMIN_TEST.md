# دليل اختبار إصلاحات Super Admin

## 🧪 اختبار سريع (5 دقائق):

### الخطوة 1️⃣: التحقق من صفحة المشتركين

```
1. افتح: http://localhost:3000/x7k9-panel-2024/tenants
2. تحقق من:
   ✓ هل تظهر قائمة المشتركين؟ (يجب ألا تكون فارغة)
   ✓ كل صف يحتوي على:
     - اسم الشركة
     - حالة (مع لون مناسب)
     - عدد السائقين
     - أيام متبقية (أو ∞ للنشط)
     - تاريخ التسجيل
     - رمز التجميد (🔒 إذا كان مجمداً)
```

### الخطوة 2️⃣: التحقق من Command Center

```
1. افتح: http://localhost:3000/x7k9-panel-2024
2. تحقق من:
   ✓ بطاقات الإحصائيات تعرض أرقام صحيحة
   ✓ جدول "الدورات المفتوحة الآن" يحتوي على:
     - المشترك (الاسم)
     - الحالة (ملونة)
     - السائقون (العدد)
     - آخر نشاط (التاريخ)
     - الإيراد (الرقم)
```

### الخطوة 3️⃣: اختبر Network Request

```
1. فتح F12 (Developer Tools)
2. اذهب للـ Network tab
3. اذهب لصفحة المشتركين
4. ابحث عن:
   - GET /api/admin/tenants
   - GET /api/admin/dashboard
5. تحقق من:
   ✓ Status: 200 (نجح)
   ✓ Response يحتوي على بيانات (partners array)
   ✓ إذا كان هناك خطأ، يعرض: { error: "...", details: "..." }
```

### الخطوة 4️⃣: اختبر الألوان حسب الحالة

في جدول الدورات المفتوحة، يجب أن تظهر الألوان:

| الحالة | اللون |
|------|------|
| تجريبي | 🟡 أصفر |
| نشط | 🟢 أخضر |
| منتهي | 🔴 أحمر |
| مجمد | ⚫ رمادي |

### الخطوة 5️⃣: اختبر عند عدم وجود بيانات

```
1. جرّب تصفية المشتركين بـ "منتهي"
2. إذا لم يكن هناك مشتركون منتهيين:
   ✓ يجب أن تظهر رسالة "لا توجد مشتركون"
   ✓ لا يجب أن تظهر قائمة فارغة بدون رسالة
```

## 🔧 استكشاف الأخطاء:

### المشكلة: صفحة المشتركين تعرض "لا توجد مشتركون"

**التشخيص:**
1. افتح Network tab
2. ابحث عن `/api/admin/tenants`
3. انظر للـ Response:
   - إذا كان `{ partners: [] }` → البيانات فعلاً فارغة
   - إذا كان `{ error: "..." }` → هناك مشكلة في الـ API

**الحل:**
- اذهب لـ Supabase console
- تحقق من أن جدول `partners` يحتوي على بيانات
- شغّل SQL:
  ```sql
  SELECT COUNT(*) FROM partners;
  ```

### المشكلة: الحالة لا تظهر في جدول الدورات

**التشخيص:**
1. افتح Network tab
2. افتح `/api/admin/dashboard` response
3. ابحث عن `open_cycles` array
4. تحقق من وجود حقول:
   - `subscription_status`
   - `status_badge`

**الحل:**
- إذا كانت هذه الحقول ناقصة، أعد تحميل الصفحة بـ Ctrl+Shift+R (Hard Refresh)
- إذا استمرت المشكلة، تأكد من أن البناء نجح: `npm run build`

### المشكلة: الواجهة تعرض البيانات لكن بدون ألوان

**التشخيص:**
1. افتح F12 → Console
2. ابحث عن أي أخطاء JavaScript

**الحل:**
- إذا كان هناك خطأ في الـ TypeScript، قد يكون المتصفح لم يحدّث الصفحة
- جرّب: Ctrl+Shift+R (Hard Refresh)

## ✅ قائمة التحقق النهائية:

- [ ] صفحة المشتركين تعرض بيانات (ليست فارغة)
- [ ] كل مشترك يحتوي على جميع المعلومات
- [ ] الحالات ملونة بشكل صحيح
- [ ] جدول الدورات المفتوحة يعرض عمود "الحالة"
- [ ] الألوان متطابقة في الموضعين
- [ ] لا توجد أخطاء في Console
- [ ] Network requests ترجع Status 200
- [ ] البيانات تتطابق بين الصفحات
